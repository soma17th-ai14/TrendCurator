# 워크플로우

이 문서는 TrendCurator의 주요 처리 워크플로우를 설명합니다.

## 정기 발행 파이프라인

```text
1. 스케줄러가 발행 서비스를 실행합니다.
2. 설정된 소스에서 AI 에이전트 관련 최신 정보를 수집합니다.
3. 원본 콘텐츠를 공통 Document 구조로 정규화합니다.
4. 공통 Document를 내부 파이프라인용 NormalizedDocument로 변환합니다.
5. 문서와 임베딩을 VectorDB에 저장합니다.
6. Digest 대상 기간의 관련 업데이트를 검색합니다.
7. 요약과 비평 메모를 생성합니다.
8. 설정된 UI 또는 채널을 통해 Daily Digest를 발행합니다.
```

## 온디맨드 질의 워크플로우

```text
1. 사용자 질문을 입력받습니다.
2. 질문을 검색 질의로 변환합니다.
3. VectorDB에서 관련 문서를 검색합니다.
4. 출처 또는 참조 기반의 근거 있는 응답을 생성합니다.
5. 프론트엔드로 답변을 반환합니다.
```

## 트렌드 비교 워크플로우

```text
1. 비교 대상 기간을 입력받습니다.
2. 각 기간에 해당하는 문서를 검색합니다.
3. 주제, 엔티티, 주장, 신호를 추출합니다.
4. 빈도, 신규성, 방향성 변화를 비교합니다.
5. 트렌드 비교 요약을 생성합니다.
6. UI에서 결과를 제공합니다.
```
