# CLAUDE.md

이 파일은 Claude가 TrendCurator 레포지토리에서 작업할 때 따라야 하는 팀 공통 지침입니다.

## 공통 문서 우선순위

1. `SKILLS.md`
2. `CLAUDE.md`
3. `AGENTS.md`
4. `CONTRIBUTING.md`
5. `README.md`
6. `docs/api-spec.md`
7. `docs/architecture.md`
8. `docs/workflow.md`

규칙이 충돌하면 `SKILLS.md`를 우선합니다.

## Claude 작업 규칙

- 모든 응답, 문서, 커밋 메시지, PR 내용은 한국어로 작성합니다.
- 사용자 요청 범위를 벗어난 구현을 하지 않습니다.
- 기존 사용자 변경사항을 임의로 삭제하거나 되돌리지 않습니다.
- 변경 전후로 관련 문서를 확인하고, 아키텍처 규칙과 충돌하지 않게 작업합니다.
- API, 서비스, 프론트엔드 연동 작업은 `docs/api-spec.md`의 계약을 먼저 확인한 뒤 진행합니다.

## 프로젝트 작업 기준

- 수집, 저장, 검색, 요약, 비교, UI 순서로 개발합니다.
- `SKILLS.md`의 폴더별 책임 경계를 지킵니다.
- RAG, VectorDB, LangGraph 관련 결정은 먼저 문서화하고 팀 리뷰를 거칩니다.
- 엔드포인트, 요청/응답 스키마, 공통 에러 코드, 주요 데이터 모델 변경은 `docs/api-spec.md`에 함께 반영합니다.
- 테스트가 필요한 변경은 테스트 없이 병합하지 않는 것을 기본 원칙으로 둡니다. 문서 전용 변경은 테스트 비대상 사유를 PR에 남깁니다.
- 변경 후에는 관련 문서 간 용어, 폴더 책임, 워크플로우 순서가 일치하는지 확인합니다.

## 커밋 및 PR 기준

- 커밋 메시지는 한국어로 작성합니다.
- PR 제목과 설명도 한국어로 작성합니다.
- PR은 작고 리뷰 가능한 단위로 유지합니다.
- PR 본문은 `CONTRIBUTING.md`의 템플릿을 사용합니다.
- PR 본문에는 변경 목적, 변경 내용, 영향 범위, 테스트 결과 또는 테스트 비대상 사유, 문서 변경 여부, 리뷰 포인트, 후속 작업을 포함합니다.
- API 계약 변경이 포함된 PR은 `docs/api-spec.md` 변경 여부와 호환성 영향을 PR 본문에 명시합니다.
- 예시 커밋 메시지: `docs: Claude 작업 지침 추가`
